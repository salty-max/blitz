# FormField

Wraps a single form control with a label and an optional hint or error. It
generates an `id`, links the label to the control (`htmlFor` plus
`aria-labelledby`), and wires `aria-describedby` and `aria-invalid` to the
message — so dropping a control inside gets you an accessible field for free.

Every form control in the library forwards these props to its root, so any of
them slots in as the direct child — `Input`, `Textarea`, `Switch`, `Checkbox`,
`Select.Trigger`, `RadioGroup`, `SegmentedControl`, `NumberStepper` and
`Combobox`.

## Hint and error

The `error` message replaces the `hint` and flags the control invalid; `required`
adds a blood asterisk to the label.

## Around other controls

A group like `RadioGroup` or `SegmentedControl` is named via `aria-labelledby`
(a `<label htmlFor>` can't point at a group); a `Combobox` takes the label on its
trigger. Each is wired the same way — drop it in as the direct child.
